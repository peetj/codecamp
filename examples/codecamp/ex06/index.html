Building a Game with Crafty JS

Back to Index

In this lesson, we are going to learn how we could go about building a 2D Game. Generally you could take one of two approaches:

  • Build the entire 2D Game yourself!
  • Use a javascript game library/engine to build upon
Which do you think is the better way? (Hint - there is no right answer to the question!)

The approach that we are going to take is to use a javascript library called CraftyJS. This will make our job a lot easier though it I wouldn't call it a walk in the park! There will still be a lot to do.

One of the advantages of CraftyJS is that it allows you to create what it call 'entities' with 'component behaviours'. So what does all THAT mean then? You can think of an entity as an object and a component as some characteristic of an object. I can create a basic entity with CraftyJS and then it allows me to add characteristics to the object by giving it a 'component' or a set of 'components'. Lets take a look at an example. The code below is not exactly CraftyJS code but it gives you the right idea. 


        /* Hey Crafty, please create me a shiny new spaceship */
        var petesSpaceShipThingy = Crafty.e();

        /* Ok that previous space ship was terrible! It doesn't do anything. Give me another!! */
        var petesUpgradedSpaceShip = Crafty.e('2D Collision')

Look at the first ship that we created in the above code. The e() method is the magic that creates the entity. However, in the first ship, we did not give it any components at all. This ship can do practically nothing!

In the second example however, we passed in 2 parameters to the e() function.These were '2D' and 'Collision'. This will create an object that has 2D capabilitites (for example we can get it's position as an (x, y) co-ordinate) and the ability to experience a Collision (this is useful if you want your ship to die when it gets shot or hit by some other object in the game).

CraftyJS gives us a whole heap of functionality that enables us to do collisions, movement with the keyboard and mouse, gravity, sprites...etc! All this simply by passing what we want to the CraftyJS Game Engine. Now you can see why using a Game Engine can make things a lot easier.

So, without further ado - let's make a game!

So, just to recap then:
  1. Games can be created from scratch or they can be built upon a library
  2. CraftyJS is one such library. You can read about it here
  3. We looked at how we might create a CraftyJS Entity (read object) with component behaviour (characteristics)
